# 들어가기 전...

QMK 를 좀 더 잘(!) 쓰려면, 한글 전환도 잘 해야 하고, 이를 위해선 결국 [[QMK - Layers]] 설정도 필요하다. 그걸 총 망라하여 여기에 정리해본다.

- QMK 키보드
- fcitx5-lua 꾸러미 및 스크립트(\~/.local/share/fcitx5/lua/imeapi/extensions/xxx.lua)

아울러, KDE 에서 영문자판은 Dvorak 이 아니고 그냥 기본자판, 즉 **Qwerty** 로 놔둔다.

# 무엇이 가능한가?

- 韓/英/日 전환 가능 : Fcitx GUI 가 아니라 QMK 와 KDE 단축키, 스크립트 이용
- 창 전환 시 초기화 : 다른 창으로 이동하면 ==영어 상태==가 된다. (이게 편한 듯.)
- 입력기 상태 기억 : 창 이동시 영어로 바뀌지만, 이전 상태는 기억한다. 예를 들어, FF 에서 한글인 상태에서 새 터미널, 즉 영어로 갔다가, 다시 FF 로 오면 다시 한글 배열로. 한/영은 확실히 되고, 일본어는 아직 완전하게 시험을 못했다. (이 작업을 위해서 `fcitx5-lua` 가 필요하다.)

# 언어간 자판 전환은 어떻게?

## QMK 측면

> [!info]
> 여긴 그냥 이론 설명이지, 뭘 해야하는 건 아니다. 이미 컴파일하여 키보드에 올려놨다.

`Shift+Space` 가 입력되면 상태에 따라 적절한 글쇠를 출력한다.
아래는 `Shift+Space`가 눌렸을 당시를 뜻한다.

- 영어 상태/Dvorak/hangul_kor(F) : 한글로 전환하기 위해 Qwerty/hangul_kor(T), 그리고 KR_HAEN 키 출력
- 한글 상태/Qwerty/hangul_kor(T) : 영어로 전환하기 위해 Dvorak/hangul_kor(F), 그리고 KC_INT4 키 출력
- 일본어 : 일본어도 위 상황과 비슷. 단, 이전 자판이 영어인지 한글인지 기억했다가 그리로 돌아간다. KC_INT2 키 출력.

> [!info] 위 용어 정리
> Dvorak/Qwerty 는 QMK 자판 배열을 뜻한다.
> hangul_kor 는 QMK 내부에서 쓰이는 변수명
> KR_HAEN : 일반 키보드의 '한영' 글쇠. (fcitx '한글' 자판으로만 전환한다.)
> KC_INT4 : 특수 글쇠. (fcitx '영문' 자판으로만 전환한다.)

## 자판 전환은 사실 명령어로.

이 방식에서 자판 전환은 fcitx 기능이 아니고, fcitx 명렁어로 한다. 다만, 이 명령을 KDE 단축키에 할당해서 실행해준다.

- 영어 : `fcitx5-remote -s keyboard-us
- 한국어 : `fcitx5-remote -s hangul`
- 일본어 : `fcitx5-remote -s mozc`

`fcitx5-remote -n` 명령으로 활성화된 입력기 이름을 확인할 수 있다.

## KDE 단축키/스크립트 할당

- 영어 / 명령어는 위 영어 항목 : ==KC_INT4 ==로 할당. 이 키는 KDE 에서 '전환'이라고 인식된다.
- 한글 / 명령어는 위 한국어 항목 : : ==KR_HAEN==으로 할당. 이 키는 '한영' 키이다.
- 일본어 / 명령어는 위 한국어 일본어 항목 : ==KC_INT2==로 할당. 이 키는 '히라가나 가타가나'로 인식된다.

---

이렇게 된 상황에서, QMK 키보드로 `Shift+Space`를 발사하면, 상황에 따라 `KC_INT4`, `KR_HAEN` 가 출력된다. 이 키는 KDE 에 단축키로 등록돼 있기 때문에, 단축키에 할당된 명렁어가 실행되어, 영문/한글 자판 전환이 이뤄진다.

일본어키는 `MO+RShift`이고, 역시 위와 같은 방식으로 작동한다.

# 프로그램별 자판 기억?

## 오작동!

Fcitx 를 쓰다보니, 창이 바뀔 때마다 영문으로 바뀌는 편이 좋지만, 이전 자판 상태는 기억하는게 작업에 편했다.
이 기능은 Fcitx 자체에서도 지원하고 있다. Reset State on focus in, 입력 방식 공유 등을 적절히 해주면 된다.
다만.. 이 때 문제는...
PC 자판은 영문 상태에서 한글로 전환이 되지만, QMK 는 계속 Dvorak 에 머물러 있고, 그 반대도 있기 때문에, Shift+Space 로 키전환을 한번 더 해야하는 귀찮은 상황이 생긴다.

QMK 로만 작업을 하면 레이어를 바꾸고, PC 에 전환키를 출력하기 때문에 이상이 없지만, PC 에선 자판 전환작업만 이뤄지고 QMK 레이어는 바뀌지 않기 때문에... '매뎌'를 보게 된다.

## 해결책은 PC 와 [[QMK - Raw HID 통신|QMK 통신]]

이 방법과, `fcitx5-lua`를 통해, 프로그램이 바뀔 때(focus in/out)를 감지하여, 자판이 바뀌어야 할 때, fcitx 로 자판을 바꿔줌과 동시에 QMK 에도 정보를 보내서 적절하게 레이어 변경을 할 수 있도록 해준다.
